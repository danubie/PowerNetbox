# Known-OK ValidateSet discrepancies.
#
# Format: <relative-path-under-Functions/>::<ParameterName>
# Lines starting with '#' and blank lines are ignored.
#
# Every entry here should have a reason — these are acknowledged exceptions
# where PowerNetbox intentionally differs from NetBox's ChoiceSets, not
# bugs waiting to be fixed.
#
# When adding a new entry, include a comment explaining WHY it is a
# legitimate deviation. When NetBox adds a value that was previously an
# exclusion, remove the entry and let the finding surface.

# ---------------------------------------------------------------------------
# Parameters whose ValidateSet is not a NetBox API enum but the script's
# heuristic matcher coincidentally picks a ChoiceSet with overlapping values.
# ---------------------------------------------------------------------------

# -Method is an HTTP verb list, not a NetBox enum. The script picks
# WebhookHttpMethodChoices by value overlap, but these are used for the
# local Invoke-RestMethod call and Netbox's bulk request dispatcher.
Helpers/InvokeNetboxRequest.ps1::Method
Helpers/Send-NBBulkRequest.ps1::Method

# -Face on Export-NBRackElevation includes 'Both' as a PowerNetbox-only
# value that means "render both sides in the SVG output", not an API filter.
DCIM/Racks/Export-NBRackElevation.ps1::Face

# -Button_Class on CustomLink is Bootstrap button variants (primary,
# secondary, success, ...). The script accidentally matches it to
# DashboardWidgetColorChoices because of overlapping colour names.
Extras/CustomLinks/New-NBCustomLink.ps1::Button_Class
Extras/CustomLinks/Set-NBCustomLink.ps1::Button_Class

# -Status on Get-NBBranch is the netbox-branching plugin's BranchStatusChoices,
# which lives outside the core NetBox source the parity tool scans. The heuristic
# matches DataSourceStatusChoices on value-overlap (new/syncing/failed). Branch
# statuses were last corrected in PR #387 (issue #385) against the plugin's
# source — see memory/PR for the 11-value authoritative list.
Plugins/Branching/Branch/Get-NBBranch.ps1::Status

# ---------------------------------------------------------------------------
# Parameters where PowerNetbox deliberately accepts backward-compat values
# that are translated to real API values in the function's begin {} block.
# ---------------------------------------------------------------------------

# Mode accepts legacy numeric/title-case values translated to API strings in begin {}.
# As of PR for #394, q-in-q (NetBox 4.2+) and its title-case variant Q-in-Q plus
# legacy code '400' are also supported via the same translation layer.
# ValidateSet values: 'Access', 'Tagged', 'Tagged All', 'Q-in-Q', 'q-in-q',
# '100', '200', '300', '400' — translate to: 'access', 'tagged', 'tagged-all', 'q-in-q'.
DCIM/Interfaces/New-NBDCIMInterface.ps1::Mode
DCIM/Interfaces/Set-NBDCIMInterface.ps1::Mode
